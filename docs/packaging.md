# Packaging & Distribution

VSView uses [PyApp](https://github.com/Varde-s-Forks/pyapp) to create standalone, self-installing executables for Windows, Linux, and macOS.

A custom CLI tool, `vsapp`, is provided to streamline the build process, icon generation, and macOS app bundling.

## Prerequisites

To build the standalone application, you need the following tools installed and available in your `PATH`:

- **[Rust & Cargo](https://rustup.rs/)**: Required to compile the `PyApp` bootstrapper.
- **[ImageMagick 7](https://imagemagick.org/)**: Required for the `vsapp icon` command to generate cross-platform icon sets.
- **[iconutil](https://developer.apple.com/library/archive/documentation/GraphicsImaging/Conceptual/DrawingWithQuartz2D/dq_images/dq_images.html)**: (**macOS only**) Required for creating App Bundles and Disk Images (`.dmg`).

## The `vsapp` CLI

The packaging logic is contained within the `vsview-pyapp` package. First, ensure the package is synced in your environment:

```bash
uv sync --package vsview-pyapp
```

Then you can run the CLI using `uv`:

```bash
uv run vsapp --help
```

### 1. Generating Icons

The `vsapp icon` command takes a source image and generates all necessary formats for Windows, macOS, and Linux.

```bash
uv run vsapp icon [OPTIONS] [SOURCE_IMAGE]
```

- **Output**: By default, icons are generated in `build/icons/`.
- **Formats**:
    - **Windows**: `vsview.ico` (multi-size icon)
    - **macOS**: `vsview.iconset/` (for `iconutil` conversion)
    - **Linux**: PNGs in standard sizes (`16x16`, `32x32`, etc.)

!!! info "Pre-generated Icons"

    The repository includes pre-generated icons in [`src/pyapp/icons/`](https://github.com/Jaded-Encoding-Thaumaturgy/vs-view/tree/main/src/pyapp/icons/).

### 2. Building the Executable

The `vsapp build` command compiles the Rust bootstrapper and embeds the Python distribution or project metadata.

```bash
uv run vsapp build [OPTIONS]
```

#### Defining the Project

You must specify what to build either via environment variables or the `.env` file:

=== "Build from PyPI (Version)"

    === "PowerShell"

        ```powershell
        powershell -NoProfile -Command { 
            $env:PYAPP_PROJECT_VERSION = "0.6.0"
            uv run vsapp build --env .\src\pyapp\.env.example
         }
        ```

    === "Bash"

        ```bash
        PYAPP_PROJECT_VERSION="0.6.0" uv run vsapp build --env src/pyapp/.env.example
        ```

=== "Build from Local Wheel"

    === "PowerShell"

        ```powershell
        powershell -NoProfile -Command {
            $env:PYAPP_PROJECT_PATH = "/home/vsview/dist/vsview-0.6.0-py3-none-any.whl"
            uv run vsapp build --env .\src\pyapp\.env.example
         }
        ```

    === "Bash"

        ```bash
        PYAPP_PROJECT_PATH="C:\vsview\dist/vsview-0.6.0-py3-none-any.whl" \
            uv run vsapp build --env src/pyapp/.env.example
        ```

### 3. macOS App Bundling & Disk Image

On macOS, applications are typically distributed as `.app` bundles or compressed Disk Images (`.dmg`). 

#### Generating the Bundle

The `vsapp bundle` command automates the creation of the standard macOS bundle structure.

```bash
uv run vsapp bundle [SOURCE_BINARY] [VERSION]
```

- **Converts** the iconset generated by `vsapp icon` into a `.icns` file using `iconutil`.
- **Creates** the standard macOS bundle structure (`Contents/MacOS`, `Contents/Resources`).
- **Generates** a valid `Info.plist`.

#### Creating a DMG Installer

To package the built `.app` bundle into a distributable Disk Image with a shortcut to the Applications folder:

```bash
# Prepare a staging directory
mkdir dmg_staging
ditto dist/AppBundle dmg_staging/VSView.app
ln -s /Applications dmg_staging/Applications

# Create the DMG
hdiutil create -volname VSView -srcfolder dmg_staging -ov -format UDZO dist/VSView.dmg
```

## Configuration

The `vsapp` tool reads configuration from a `.env` file. You can use [`src/pyapp/.env.example`](https://github.com/Jaded-Encoding-Thaumaturgy/vs-view/tree/main/src/pyapp/.env.example) as a template.

## Linux Installation

For Linux distributions, a helper script is provided in [`src/pyapp/install.sh`](https://github.com/Jaded-Encoding-Thaumaturgy/vs-view/tree/main/src/pyapp/install.sh). 

The installation script will:

1. Move the binary to `~/.local/bin`.
2. Install icons of various sizes to `~/.local/share/icons/hicolor`.
3. Create a `.desktop` entry for application menu integration.

```bash
# Run from the directory containing the binary and icons/
bash install.sh
```
